# Phase 0 산출물: 회원가입 수집 항목 · 개인정보처리방침 개정안 · 동의 문구

> 상태: 초안 (검토용) · 작성일: 2026-07-24
> 관련: `docs/PRD-membership-crm.md` Phase 0
> 주의: 이 문서는 **설계 초안**이며, 실제 `app/(site)/privacy/page.tsx`(라이브 방침)에는
> Phase 2(로그인 구현) 배포 시점에 맞춰 반영한다. 지금 라이브 페이지를 먼저 바꾸면

## 1. 확정된 수집 항목

사용자 결정 사항을 반영한 최종 범위:

| 구분 | 항목 | 수집 시점 | 비고 |
|---|---|---|---|
| 필수(가입) | 이메일, 이름, 전화번호 | 회원가입 시 | 이메일/비밀번호 가입 기준. 전화번호는 본인확인이 아닌 연락처 목적 |
| 필수(소셜) | 이메일, 이름, 소셜 프로필 식별자 | 소셜 로그인 시 | 카카오/네이버/구글이 제공하는 범위 내에서만 수신 |
| 선택 | 마케팅 수신 동의 여부(이메일/SMS/카카오톡 알림) | 가입 시 또는 마이페이지에서 | 미동의해도 가입 가능해야 함(정보통신망법) |
| 자동 생성 | 접속 로그, IP, 기기 정보 | 서비스 이용 중 | 기존 방침 1항과 동일, 회원 여부와 무관하게 이미 수집 중 |
| 결제 연동 | Stripe Customer ID | 로그인 후 결제 시 | 카드정보 등 원본 결제정보는 기존과 동일하게 Stripe가 직접 처리, 사이트는 저장하지 않음 |
| 연령 확인 | 만 14세 이상 여부(체크) | 가입 시 | 만 14세 미만 가입 차단 |

**저장하지 않는 것**: 카드번호 등 결제 원본 정보(Stripe 위탁), 비밀번호 평문(해시만 저장).

## 2. 개인정보처리방침 개정안 (반영 대상: `app/(site)/privacy/page.tsx`)

기존 방침은 "회원가입 없이 열람"을 전제로 작성되어 있어, 회원 기능 추가 시 다음과
같이 개정한다. **기존 조항은 유지**하고 회원 관련 내용만 추가/수정하는 방식이다
(기존 비회원 열람 흐름은 그대로 두므로 1~2항의 큰 틀은 손대지 않는다).

### 1. 수집하는 개인정보 항목 — 추가 문단

> `${siteConfig.name}는 회원가입 없이도 대부분의 콘텐츠를 열람할 수 있습니다. 다만
> 회원가입, 소셜 로그인, 마케팅 정보 수신에 동의하는 경우 다음 정보를 수집합니다.`
>
> - (필수) 이메일, 이름, 전화번호 — 이메일 가입 시
> - (필수) 이메일, 이름 — 소셜 로그인(카카오, 네이버, 구글) 시, 각 제공자가 전달하는
>   범위 내에서만 수신
> - (선택) 마케팅 정보 수신 동의 여부 및 수신 채널(이메일/SMS/카카오톡 알림)
> - (자동) 로그인 세션 정보, 최근 로그인 일시
> - (편집샵 이용 시, 회원) 구매 이력은 Stripe에 저장된 정보를 연동 조회하며 사이트
>   DB에는 결제 식별자(Customer ID)만 보관합니다.

### 3. 개인정보의 이용 목적 — 항목 추가

> - 회원 식별 및 로그인 유지
> - (선택 동의 시) 신규 콘텐츠·상품 소식 등 마케팅 정보 안내
> - 향후 서비스 이용 패턴에 기반한 맞춤형 콘텐츠·상품 추천(타겟팅) — **이 항목은
>   광고·타겟팅 기능이 실제로 출시되는 시점에 방침에 활성화하며, Phase 0~1 시점에는
>   포함하지 않는다.** (수집 목적 명시는 실제 이용 시점과 함께 가야 함)

### 4. 개인정보의 보유 및 이용 기간 — 항목 추가

> - 회원 정보는 **회원 탈퇴 시 지체 없이 파기**한다. 단, 관계 법령이 정한 경우(전자상거래법상
>   계약·결제 기록 등)에는 해당 기간 동안 분리 보관 후 파기한다.
> - 마케팅 동의 이력은 동의/철회 여부를 증빙하기 위해 **탈퇴 후 3년간** 별도 보관 후
>   파기한다(분쟁 대응 목적, 이용에는 사용하지 않음).

### 5. 개인정보의 제3자 제공 및 처리 위탁 — 항목 추가

| 수탁업체 | 위탁 업무 |
|---|---|
| 카카오(주) | 카카오 소셜 로그인 |
| ㈜네이버 | 네이버 소셜 로그인 |
| Google LLC | 구글 소셜 로그인 |
| (DB 제공자 — Phase 1 결정) | 회원 정보 저장 |
| Stripe, Inc. | 결제 처리 (기존과 동일) |

### 6. 이용자의 권리 — 항목 추가

> - 회원은 마이페이지에서 언제든지 **마케팅 수신 동의를 철회**할 수 있으며, 철회는
>   즉시 반영됩니다.
> - 회원은 언제든지 **회원 탈퇴**를 요청할 수 있으며, 탈퇴 시 개인정보는 4항의 기준에
>   따라 파기됩니다.
> - 만 14세 미만은 가입할 수 없으며, 확인 시 즉시 가입이 제한됩니다.

## 3. 가입 화면 동의 문구 (UI 카피 초안)

Phase 2(인증 구현)에서 회원가입 폼에 그대로 사용할 수 있는 문구.

```
[필수] 만 14세 이상입니다.                                    [ ]

[필수] 이용약관에 동의합니다.                    [보기]        [ ]
[필수] 개인정보 수집·이용에 동의합니다.          [보기]        [ ]

[선택] 이메일/SMS/카카오톡으로 신규 콘텐츠 및    [보기]        [ ]
       상품 소식을 받아보시겠습니까?

※ 선택 동의를 하지 않아도 회원가입 및 서비스 이용에 제한이 없습니다.
```

- "[보기]" 링크는 각각 이용약관 페이지 / 개인정보처리방침(2장 개정 조항) / 마케팅
  동의 상세 안내로 연결한다.
- 마케팅 동의 체크박스는 **기본값 미체크**(옵트인)로 시작해야 한다(정보통신망법
  제50조 — 사전 동의 원칙).
- 만 14세 확인은 별도 체크박스로 분리(단순 "이용약관 동의"에 묶지 않음) — 추후 실제
  생년월일 입력 검증으로 고도화할 수 있으나 Phase 0~2 범위에서는 체크박스로 충분.

### 마케팅 동의 철회/변경 문구 (마이페이지용)

```
마케팅 정보 수신    [이메일 ✓]  [SMS ✓]  [카카오톡 알림 ✓]
                                                    [저장]

수신 동의를 해제해도 회원 서비스 이용에는 영향이 없습니다.
```

## 4. 아직 남은 것 (Phase 0 범위 밖)

- **이용약관 전문**: 위 동의 문구가 참조하는 "이용약관"은 별도 법률 문서로,
  전자상거래(편집샵)를 포함하는 서비스 약관 전문 작성이 필요하다. 이는 Phase 0의
  "개인정보처리방침" 범위를 넘어서므로, 회원가입 UI를 만드는 **Phase 2 착수 전에**
  별도로 준비가 필요하다는 점만 여기 남겨둔다.
- **DB 제공자 확정**: 5장 위탁 표의 "DB 제공자"는 Phase 1에서 Vercel Postgres vs
  Supabase 결정 후 채워 넣는다.
